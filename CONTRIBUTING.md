# Contributing

> **Status: placeholder.** This project does not yet accept external contributions. This file exists to state the rule up front, before any pull request shows up, not to serve as a signed legal agreement — see the disclaimer at the bottom.

## Why a CLA (Contributor License Agreement) will be required

This project is dual-licensed: the code is public under **AGPL-3.0** (see `LICENSE`), and a separate commercial license is planned for parties who can't accept AGPL's obligations (see `COMMERCIAL-LICENSE.md`). Offering that commercial license is only possible if the maintainer holds full rights to the entire codebase — a contribution accepted without a CLA would mean the maintainer can't relicense that piece commercially, breaking the dual-licensing model for the whole project.

Because of that, **any external contribution will require signing a CLA before it can be merged**, once one is in place.

## Current state

- No CLA process exists yet — because there have been no external contributors to apply it to.
- A real CLA (the actual legal document contributors would sign) will be drafted with legal review the first time an external contribution is actually proposed, not before.
- Until then, treat this repo as maintainer-only for merged code.

If you're reading this because you want to contribute: open an issue first to discuss it. The CLA will get drafted for real at that point.
